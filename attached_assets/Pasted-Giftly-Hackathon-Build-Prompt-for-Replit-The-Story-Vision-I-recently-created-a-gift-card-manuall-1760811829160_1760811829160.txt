Giftly - Hackathon Build Prompt for Replit
The Story & Vision
I recently created a gift card manually for a friend before his birthday lunch. He was going to get his BMW X5 washed at his favorite local car wash before meeting us. I used ChatGPT to combine an image of his BMW with the car wash's branding, then embedded a link to a virtual card I'd bought him. It worked, but it was super clunky and required a lot of hand-holding.
The Vision: What if creating a personalized gift card for ANY local merchant was as easy as buying a Starbucks gift card online?
That's Giftly. You paste a local business URL, AI generates a beautiful branded gift card, you load it with real money via Stripe, and the recipient gets it in their Apple/Google Pay wallet instantly.

Core Features to Build
1. Landing Page

Hero section explaining Giftly
"Create a Gift" CTA button
Clean, modern design

2. Gift Creation Flow

URL input for local business
"Analyze" button that calls Claude API
Claude analyzes the business webpage and extracts:

Business name
Business type (car wash, bakery, coffee shop, etc.)
Brand colors
Vibe/aesthetic
What they sell/do


Display beautiful gift card preview with branding
Gift value selector ($25, $50, $100, custom)
Personal message input (optional)
Recipient name and phone number inputs
"Purchase & Send" button

3. Payment Flow

Stripe Checkout integration
After successful payment, create Stripe Issuing virtual card
Generate unique gift link (giftly.app/gift/[unique-id])
Show QR code for easy mobile access
Show success message with shareable link

4. Gift View Page (Recipient Experience)

Display beautiful branded gift card
Show gift amount and personal message
Show recipient name
Detect device (iOS vs Android)
Show "Add to Apple Pay" or "Add to Google Pay" button
Handle wallet provisioning via Stripe API
Confetti animation on successful add
Show card details (number, expiry, CVV) as fallback

5. Backend API Endpoints
POST /api/analyze-business
- Body: { url: string }
- Fetches business webpage
- Sends to Claude for analysis
- Returns: { businessName, type, colors, vibe, description }

POST /api/create-checkout
- Body: { amount, recipientName, recipientPhone, recipientEmail, message, businessInfo }
- Creates Stripe Checkout session
- Returns: { sessionId, checkoutUrl }

POST /api/webhook
- Handles Stripe webhook for successful payment
- Creates Stripe Issuing cardholder
- Creates Stripe Issuing virtual card with spending limit
- Stores gift details with unique ID
- Returns success

GET /api/gift/:giftId
- Returns gift details including card info

POST /api/provision-wallet
- Body: { cardId, walletType: 'apple_pay' | 'google_pay' }
- Calls Stripe Issuing API to provision card to wallet
- Returns provisioning data
What to Simplify/Skip for Demo
Skip:

User authentication/login
Database (use in-memory storage)
Email notifications
SMS (use QR code + shareable link instead)
Gift history/tracking
Real web scraping (just fetch URL and extract what you can)

Simplify:

Card design: Use Claude to suggest colors/theme, render with React + Tailwind
No image generation: Use emoji/icons + gradient backgrounds
Manual card details reveal: Show card number, CVV, expiry if wallet provisioning has issues

Stripe Issuing:
javascript// Create cardholder
const cardholder = await stripe.issuing.cardholders.create({
  name: recipientName,
  email: recipientEmail,
  phone_number: recipientPhone,
  billing: { address: { line1: '123 Main St', city: 'Austin', state: 'TX', postal_code: '78701', country: 'US' } },
  type: 'individual'
});

// Create virtual card with spending limit
const card = await stripe.issuing.cards.create({
  cardholder: cardholder.id,
  currency: 'usd',
  type: 'virtual',
  status: 'active',
  spending_controls: {
    spending_limits: [{ amount: giftAmount * 100, interval: 'all_time' }]
  }
});

// Provision to Apple Pay
const provisionData = await stripe.issuing.cards.deliverCard(card.id, {
  type: 'digital_wallet',
  digital_wallet: { type: 'apple_pay' }
});
